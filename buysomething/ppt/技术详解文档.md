# 优选购物 — 技术详解文档（大白话版）

> 这份文档把项目里每一个技术细节都用最简单的话解释清楚，帮助你真正理解每一行代码背后的原理。

---

## 一、项目整体架构（一句话版）

**微信小程序（前端）+ 腾讯云 CloudBase（后端）= 不用自己搭服务器就能跑的全栈电商**

传统开发：你要自己买服务器、装数据库、写接口。  
云开发：腾讯帮你搞定了服务器和数据库，你只管写代码调接口。

## 二、前端技术（小程序部分）

### 2.1 小程序四大件

每个页面都有 4 个文件：

| 文件 | 干什么的 | 类比 |
|------|---------|------|
| `.wxml` | 页面结构，决定页面长什么样 | HTML |
| `.wxss` | 页面样式，决定颜色、大小、位置 | CSS |
| `.js` | 页面逻辑，决定点击按钮干什么 | JavaScript |
| `.json` | 页面配置，比如标题栏文字 | 配置文件 |

### 2.2 页面跳转

小程序有 3 种跳转方式：

- `wx.navigateTo`：打开新页面，可以返回（压栈）
- `wx.redirectTo`：替换当前页面，不能返回
- `wx.switchTab`：切换底部 Tab 栏页面（首页/分类/购物车/我的）

**为什么下单用 navigateTo？** 因为用户可能想返回改地址。  
**为什么登录成功用 redirectTo？** 因为登录页不需要再回来了。

### 2.3 数据绑定

WXML 里用 `{{变量名}}` 显示数据，JS 里用 `this.setData({ 变量名: 值 })` 更新数据，页面自动刷新。

```
// WXML
<text>{{product.name}}</text>

// JS
this.setData({ product: { name: 'T恤' } })
// 页面自动显示 "T恤"
```

**原理**：小程序有一套数据监听机制，setData 时会对比新旧数据，只更新变化的部分（虚拟 DOM 的简化版）。

### 2.4 列表渲染

用 `wx:for` 循环显示列表：

```xml
<view wx:for="{{products}}" wx:key="_id">
  <text>{{item.name}}</text>
</view>
```

`wx:key` 是必须的，告诉小程序每条数据的唯一标识，这样更新列表时只重新渲染变化的那一条，而不是整个列表。

### 2.5 条件渲染

- `wx:if`：条件为 true 才渲染这个元素（完全不存在于页面中）
- `hidden`：元素始终存在，只是隐藏/显示

**什么时候用哪个？** 频繁切换用 `hidden`（比如 tab 切换），不频繁用 `wx:if`（比如空状态提示）。

### 2.6 级联选择器（地址选择）

省→市→区 三级联动，用小程序的 `<picker>` 组件：

```
<picker mode="region" bindchange="onRegionChange">
  {{region[0]}} {{region[1]}} {{region[2]}}
</picker>
```

用户选完省后，市的选项自动变成该省下的城市，选完市后区的选项自动变成该市下的区。

**原理**：微信内置了中国行政区划数据，`mode="region"` 直接调用，不用自己维护省市区数据。

### 2.7 本地存储

```
wx.setStorageSync('key', value)  // 存
wx.getStorageSync('key')         // 取
wx.removeStorageSync('key')      // 删
```

**我们存了什么？**
- `userInfo`：用户信息（昵称、手机号）
- `merchantInfo`：商家信息
- `adminInfo`：管理员信息
- `cart`：购物车数据
- `isLogin`：登录状态

**为什么不全存数据库？** 购物车这类频繁读写的数据放本地更快，不依赖网络。登录状态放本地是为了不用每次打开小程序都重新登录。

## 三、后端技术（云函数部分）

### 3.1 云函数是什么？

云函数 = 跑在腾讯云上的 JavaScript 函数。你写好代码上传，腾讯帮你运行。

**和传统后端的区别：**
- 传统：服务器 24 小时运行，不管有没有请求都在花钱
- 云函数：有请求才运行，按调用次数收费（免费额度足够用）

### 3.2 云函数怎么写？

```javascript
const cloud = require('wx-server-sdk')
cloud.init({ env: '你的环境ID' })
const db = cloud.database()

exports.main = async (event, context) => {
  // event 就是前端传来的参数
  // context 包含调用者信息
  const { action } = event
  
  if (action === 'list') {
    const result = await db.collection('products').get()
    return { code: 0, data: result.data }
  }
}
```

**解释：**
- `cloud.init()`：连接到你的云环境
- `db.collection('products')`：选择数据库中的 products 集合
- `.get()`：查询所有数据
- `return`：返回给前端

### 3.3 前端怎么调用？

```javascript
const res = await wx.cloud.callFunction({
  name: 'products',     // 云函数名
  data: { action: 'list' }  // 传的参数
})
console.log(res.result) // 云函数 return 的内容
```

### 3.5 数据库操作基础

```javascript
// 查询
db.collection('products').where({ status: 1 }).get()

// 添加
db.collection('products').add({ data: { name: 'T恤', price: 79 } })

// 更新
db.collection('products').doc('商品ID').update({ data: { price: 59 } })

// 删除
db.collection('products').doc('商品ID').remove()

// 排序
db.collection('products').orderBy('sales', 'desc').get()

// 分页
db.collection('products').skip(10).limit(10).get()  // 第2页，每页10条
```

### 3.6 服务端日期

```javascript
db.serverDate()  // 数据库服务器当前时间
```

**为什么不用 `new Date()`？** 因为用户手机时间可能不准，用服务器时间保证一致性。

## 四、数据库设计

### 4.1 NoSQL vs SQL

我们用的是 NoSQL（文档数据库），和 SQL 的区别：

| 特性 | NoSQL (我们用的) | SQL |
|------|----------------|-----|
| 存储方式 | 像JSON文档 | 像Excel表格 |
| 字段 | 每条数据可以不一样 | 必须预先定义列 |
| 关系 | 不支持JOIN | 支持表关联 |
| 适合 | 快速开发、灵活数据 | 严格结构、复杂查询 |

### 4.3 集合权限

所有集合都设为 `PRIVATE` 或 `ADMINWRITE`：

- `PRIVATE`：只有创建者能读写（实际只通过云函数操作，云函数有管理员权限）
- `ADMINWRITE`：所有人可读，仅管理员可写

**为什么不设为所有人可写？** 防止用户绕过云函数直接操作数据库（比如改价格、刷订单）。

## 五、核心功能详解

### 5.1 密码加密（MD5 + Salt）

```
原始密码: 123456
盐值(Salt): 随机字符串 x9k2m
加密后: MD5("123456" + "x9k2m") = "a1b2c3d4..."
```

**为什么不直接 MD5？** 因为 MD5 可以被彩虹表反查（网上有 MD5 数据库）。加了随机盐值后，同样的密码加密结果不一样，彩虹表就没用了。

**流程：**
1. 注册时：生成随机盐 → 密码+盐 → MD5 → 存盐和MD5值
2. 登录时：取出盐 → 输入密码+盐 → MD5 → 和存储的MD5比对

### 5.2 AI 客服（DeepSeek 接入）

**流程：**
1. 用户从商品详情页点"客服"
2. 前端把商品信息（名称、价格、规格、描述）传给云函数
3. 云函数构建"系统提示词"（告诉AI你是什么角色、当前商品信息）
4. 调用 DeepSeek API，传对话历史 + 当前问题
5. 返回 AI 回答

**关键代码逻辑：**
```javascript
// 系统提示词 = 角色设定 + 商品上下文
let systemPrompt = `你是优选购物的智能客服小优。
当前商品：${productInfo.name}
价格：${productInfo.price}元
颜色：${productInfo.specs.colors.join('、')}`

// 对话历史（最近6条，控制token消耗）
const messages = [
  { role: 'system', content: systemPrompt },
  ...最近6条历史对话,
  { role: 'user', content: 用户当前问题 }
]

// 调用 API
const result = await callDeepSeekAPI(messages)
```

**为什么商品要绑定到 AI？** 这样用户说"这个多少钱"，AI 就知道是哪个商品，不用反问"您说的是哪件商品"。

### 5.3 商家实时聊天

**核心设计：conversationId**

每个会话有唯一 ID = `商家ID_用户ID_商品ID`，这样：
- 同一个商家和同一个用户聊不同商品，是不同会话
- 不同用户和同一个商家聊，是不同会话

**消息流转：**
1. 用户发消息 → 存入 chat_messages 集合 → 更新会话摘要
2. 商家端 5 秒轮询 → 查询新消息 → 显示
3. 商家回复 → 存入同一集合 → 用户端 5 秒轮询看到

**为什么用轮询不用 WebSocket？** 微信小程序的 WebSocket 有连接数限制，且云开发不原生支持。5 秒轮询对于客服场景够用了。

### 5.4 物流模拟系统

**核心算法：**

1. 获取商家地址坐标（发货地）和用户地址坐标（收货地）
2. 画一条直线连接两地
3. 检测直线是否穿过海/湖区域
4. 如果穿过，计算绕行路径点
5. 按天推进货物位置（模拟运输进度）

**避海算法：**
```
1. 定义海/湖的矩形区域（渤海、黄海等）
2. 检测直线和矩形是否有交集
3. 如果有交集，在矩形边缘找绕行点
4. 路线变成：起点 → 绕行点 → 终点
```

**自动签收：**
- 发货后记录 `shipTime`
- 每次查看物流时检查：当前时间 - shipTime > 15天？
- 超过15天自动将订单状态改为3（已收货）

### 5.5 售后三方仲裁

**流程：**
```
用户申请售后 → status: pending
    ↓
商家审核：
  同意 → status: approved（退款/退货完成）
  拒绝 → status: rejected（进入管理员裁定）
    ↓
管理员裁定：
  同意 → status: admin_approved（强制退款）
  驳回 → status: admin_rejected（售后关闭）
```

**为什么需要三方？** 防止商家恶意拒绝合理的售后请求，管理员作为第三方公正裁定。

### 5.6 评价系统

```
用户提交评价 → 存入 reviews 集合
  → 重新计算该商品的平均评分
  → 更新 products 集合的 rating 和 reviewCount
```

### 5.7 角色一键切换

不用重新登录，直接修改本地存储和全局数据：

```javascript
switchToUser() {
  // 直接绑定测试用户
  wx.setStorageSync('userInfo', testUser)
  wx.setStorageSync('isLogin', true)
  app.globalData.isLogin = true
  app.globalData.userInfo = testUser
  wx.switchTab({ url: '/pages/profile/profile' })
}
```

**为什么能这样？** 因为三个角色的认证信息是独立的（userInfo / merchantInfo / adminInfo），互不干扰。

## 六、安全体系

### 6.1 密码安全
- MD5 + Salt 哈希存储
- 数据库中永远不存明文密码

### 6.2 数据库安全
- 所有集合设为 PRIVATE/ADMINWRITE
- 前端不能直接操作数据库，必须通过云函数

### 6.3 CI/CD 安全
- GitHub Actions 自动构建
- Gitleaks 检测代码中是否泄露密钥
- ESLint 检查代码规范

### 6.4 云函数权限
- 云函数以管理员身份运行，能读写所有集合
- 但云函数代码中会校验用户身份（比如商家只能删自己的商品）

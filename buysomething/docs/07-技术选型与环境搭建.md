## 一、项目概述

### 1.1 项目简介

优选购物小程序是一款基于微信小程序平台的电商购物应用，为用户提供便捷的移动端购物体验。项目采用前后端分离架构，前端使用微信小程序原生开发，后端使用腾讯云CloudBase云开发平台。

### 1.2 核心功能

- **用户模块**: 用户注册、登录、个人信息管理、收货地址管理
- **商品模块**: 商品浏览、搜索、分类、详情展示、商品推荐
- **购物车模块**: 添加商品、修改数量、删除商品、购物车结算
- **订单模块**: 创建订单、订单列表、订单详情、订单支付、订单状态管理
- **营销模块**: 优惠券、限时秒杀、今日特价、签到有礼

### 1.3 技术特点

- **Serverless架构**: 无需管理服务器，按需付费
- **云原生**: 利用云开发平台提供的完整解决方案
- **前后端分离**: 小程序前端 + CloudBase后端
- **自动扩缩容**: 支持高并发访问
- **高可用性**: 依赖腾讯云基础设施，自动容灾备份

## 二、技术选型

### 2.1 前端框架选择及理由

#### 2.1.1 选择方案

| 技术栈 | 版本 | 用途 |
|--------|------|------|
| 微信小程序原生框架 | 基础库 2.19.4+ | 前端开发框架 |
| WXML | - | 页面结构标记语言 |
| WXSS | - | 页面样式语言 |
| JavaScript/TypeScript | ES6+ | 页面逻辑语言 |
| wx.cloud SDK | 内置 | 云开发SDK |

#### 2.1.2 选择理由

**✅ 选择微信小程序原生框架的理由:**

1. **官方支持与稳定性**
   - 微信官方持续维护更新，技术成熟稳定
   - 丰富的官方文档和开发者社区支持
   - 与微信生态深度集成，API调用更加便捷

2. **性能优势**
   - 原生渲染，性能优于H5和uni-app等跨平台方案
   - 更小的包体积，加载速度更快
   - 更好的用户体验和流畅度

3. **开发效率**
   - 无需配置复杂的构建工具和开发环境
   - 微信开发者工具提供完整的开发、调试、预览功能
   - 热重载、实时预览，提高开发效率

4. **功能完整性**
   - 完整的微信支付、分享、登录等原生能力
   - 支持云开发，无需搭建后端服务器
   - 丰富的组件库和API接口

5. **用户触达**
   - 微信月活用户超过13亿，用户基数庞大
   - 小程序无需下载安装，即用即走
   - 微信生态内的社交传播优势

**❌ 未选择其他方案的原因:**

| 方案 | 不选择原因 |
|------|-----------|
| **uni-app** | 虽然支持多端，但性能不如原生，包体积更大 |
| **Taro** | 学习成本高，需要配置复杂的编译环境 |
| **mpvue** | 已经停止维护，不适合新项目开发 |
| **wepy** | 官方已停止维护，不再推荐使用 |

#### 2.1.3 技术栈详解

**1. WXML (WeiXin Markup Language)**
- 作用: 定义小程序页面结构
- 特点: 类似HTML，但标签有限，组件化开发
- 优势: 与微信小程序运行时深度集成，性能最优

**2. WXSS (WeiXin Style Sheets)**
- 作用: 定义小程序页面样式
- 特点: 类似CSS，支持rpx响应式单位
- 优势: 自动适配不同屏幕尺寸，开发效率高

**3. JavaScript (ES6+)**
- 作用: 实现页面逻辑和交互
- 特点: 支持ES6+语法，箭头函数、Promise、async/await
- 优势: 代码简洁，易维护，异步处理更优雅

**4. wx.cloud SDK**
- 作用: 调用云开发能力
- 特点: 内置在小程序基础库中，无需额外引入
- 优势: 无缝集成云函数、云数据库、云存储

   - 良好的错误处理机制

#### 2.2.4 数据库技术栈

**选择文档数据库的理由:**

1. **灵活的数据模型**
   - JSON文档格式，无需预定义Schema
   - 支持嵌套数据结构
   - 适合电商场景的复杂商品数据

2. **查询能力强大**
   - 支持复杂查询条件
   - 支持聚合操作
   - 支持地理位置查询

3. **水平扩展**
   - 自动分片，支持海量数据
   - 读写分离，性能优化
   - 实时数据同步

4. **安全规则**
   - 细粒度的权限控制
   - 基于用户OpenID的数据隔离
   - 保护用户隐私数据

### 2.3 数据库选择及理由

#### 2.3.1 选择方案

| 数据库类型 | 版本 | 特点 |
|-----------|------|------|
| CloudBase文档数据库 | NoSQL | 主数据库，存储业务数据 |
| 本地存储 | wx.storage | 缓存用户OpenID和临时数据 |

#### 2.3.2 选择文档数据库的理由

**✅ 选择文档数据库(NoSQL)的优势:**

1. **适合电商场景**
   - 商品数据结构复杂，文档数据库更灵活
   - 支持动态字段，便于扩展商品属性
   - 适合存储商品的多规格、多图片等信息

2. **开发效率高**
   - 无需定义Schema，快速迭代
   - JSON格式，与前端数据格式一致
   - 减少ORM层，代码更简洁

3. **性能优秀**
   - 读写性能高，适合高并发场景
   - 支持索引优化，查询速度快
   - 自动缓存，减少数据库压力

4. **成本效益**
   - 免费额度充足(2GB存储 + 5万次读写/天)
   - 按量付费，用量可控
   - 无需运维成本

**❌ 未选择关系型数据库(MySQL)的理由:**

| 不选择原因 | 说明 |
|-----------|------|
| **Schema变更频繁** | 电商业务迭代快，表结构修改成本高 |
| **复杂关联查询** | 商品属性多，关联查询复杂，性能差 |
| **扩展性差** | 需要手动分表分库，运维复杂 |
| **成本较高** | CloudBase MySQL需要单独付费 |

#### 2.3.3 数据库集合设计

根据业务需求，设计了以下10个数据库集合:

| 集合名称 | 用途 | 主要字段 |
|---------|------|---------|
| `users` | 用户信息 | openid, phone, nickname, avatar, memberLevel |
| `products` | 商品信息 | name, price, stock, images, categoryId |
| `categories` | 商品分类 | name, icon, parentId, sort |
| `cart` | 购物车 | openid, productId, quantity, spec |
| `orders` | 订单信息 | orderId, openid, status, totalAmount |
| `order_items` | 订单明细 | orderId, productId, price, quantity |
| `addresses` | 收货地址 | openid, name, phone, address, isDefault |
| `coupons` | 优惠券 | name, type, value, minAmount, startTime |
| `favorites` | 收藏 | openid, productId, createTime |
| `banners` | Banner轮播 | imageUrl, linkUrl, sort, status |

#### 2.3.4 数据库安全规则

```
用户数据集合 (users, cart, orders, addresses, favorites):
  读取权限: auth.openid == doc.openid  # 仅用户本人可读
  写入权限: auth.openid == doc.openid  # 仅用户本人可写

公共数据集合 (products, categories, banners, coupons):
  读取权限: true                        # 所有人可读
  写入权限: false                       # 禁止前端写入(通过云函数)
```

#### 3.1.5 配置app.js

```javascript
App({
  onLaunch(options) {
    console.log('小程序启动', options)

    // 初始化CloudBase
    if (wx.cloud) {
      wx.cloud.init({
        env: 'buysomething-6gbmbtpxff05be35',
        traceUser: true
      })
      console.log('CloudBase 初始化成功')
    } else {
      console.error('请使用 2.2.3 或以上的基础库以使用云能力')
    }

    this.checkLoginStatus()
  },

  onShow(options) {
    // 小程序显示
  },

  onHide() {
    // 小程序隐藏
  },

  checkLoginStatus() {
    const userInfo = wx.getStorageSync('userInfo')
    const isLogin = wx.getStorageSync('isLogin')
    
    if (userInfo && isLogin) {
      this.globalData.isLogin = true
      this.globalData.userInfo = userInfo
      console.log('用户已登录:', userInfo)
    } else {
      this.globalData.isLogin = false
      this.globalData.userInfo = null
    }
  },

  login(userInfo) {
    this.globalData.isLogin = true
    this.globalData.userInfo = userInfo
    wx.setStorageSync('userInfo', userInfo)
    wx.setStorageSync('isLogin', true)
  },

  logout() {
    this.globalData.isLogin = false
    this.globalData.userInfo = null
    wx.removeStorageSync('userInfo')
    wx.removeStorageSync('isLogin')
  },

  getDB() {
    if (wx.cloud) {
      return wx.cloud.database()
    }
    return null
  },

  globalData: {
    isLogin: false,
    userInfo: null,
    cartCount: 0,
    primaryColor: '#FF6B6B',
    secondaryColor: '#4ECDC4'
  }
})
```

#### 3.3.4 数据库操作示例

```javascript
// 在页面中使用数据库
Page({
  onLoad() {
    this.getProducts()
  },

  // 获取商品列表
  getProducts() {
    const db = wx.cloud.database()
    
    db.collection('products')
      .where({
        status: 1
      })
      .orderBy('createTime', 'desc')
      .limit(20)
      .get()
      .then(res => {
        console.log('商品列表:', res.data)
        this.setData({
          products: res.data
        })
      })
      .catch(err => {
        console.error('查询失败:', err)
      })
  },

  // 添加购物车
  addToCart(product) {
    const app = getApp()
    const db = wx.cloud.database()
    
    db.collection('cart').add({
      data: {
        openid: app.globalData.userInfo.openid,
        productId: product._id,
        name: product.name,
        price: product.price,
        image: product.images[0],
        quantity: 1,
        createTime: new Date()
      }
    }).then(res => {
      wx.showToast({
        title: '添加成功',
        icon: 'success'
      })
    }).catch(err => {
      console.error('添加失败:', err)
    })
  }
})
```

## 五、常见问题

### 5.1 环境配置问题

**问题1: 提示"请使用 2.2.3 或以上的基础库以使用云能力"**

```
解决方法:
1. 检查微信开发者工具版本
2. 点击"详情" → "本地设置" → 勾选"不校验合法域名"
3. 更新基础库版本到2.2.3或以上
```

**问题2: 云函数调用失败，提示"云函数不存在"**

```
解决方法:
1. 检查云函数是否已部署
2. 检查云函数名称是否正确
3. 重新部署云函数
```

**问题3: 数据库查询失败**

```
解决方法:
1. 检查环境ID是否正确
2. 检查集合是否已创建
3. 检查安全规则是否配置正确
4. 检查用户是否已登录(需要OpenID)
```

### 5.2 开发调试问题

**问题1: TypeError: Cannot read property '__subPageFrameEndTime__' of null**

```
解决方法:
1. 检查页面生命周期中是否有未正确销毁的定时器
2. 在onUnload中清除所有定时器
3. 重启微信开发者工具
4. 清除缓存后重新编译
```

**问题2: 页面跳转失败**

```
解决方法:
1. 检查页面路径是否在app.json中注册
2. 检查跳转方法是否正确(switchTab/navigateTo)
3. 检查页面路径是否正确
```

**问题3: 样式不生效**

```
解决方法:
1. 检查.wxss文件名是否正确
2. 检查样式选择器是否正确
3. 检查是否有样式覆盖
```

\###2. 技术选型

前端框架选择及理由

| 技术栈 | 版本 | 用途 |

|--------|------|------|

| 微信小程序原生框架 | 基础库 2.19.4+ | 前端开发框架 |

| WXML | - | 页面结构标记语言 |

| WXSS | - | 页面样式语言 |

| JavaScript/TypeScript | ES6+ | 页面逻辑语言 |

| wx.cloud SDK | 内置 | 云开发SDK |
选取理由：可以采用云后端，解决后端问题，操作容易，维护简单，性能好，功能完整，并且后续开发可以很方便的加入微信支付这个跟手机商城密不可分的功能



后端框架选择及理由

| 技术栈 | 版本 | 用途 |

|--------|------|------|

| 腾讯云CloudBase | - | 云开发平台 |

| 云函数 | Node.js 16.x | 服务端业务逻辑 |

| 云数据库 | 文档数据库(类MongoDB) | 数据存储 |

| 云存储 | 对象存储 | 文件存储 |

| 云接入 | HTTP API | API网关 |

理由：开发效率高 ，开发便捷，链接方便，读写性能高， 支持索引优化，查询速度快，自动缓存，减少数据库压力，部署容易。

# 注册功能部署指南

## 概述

本文档说明如何部署注册功能所需的CloudBase云函数。

## 已创建的文件

### 1. 注册页面
- `pages/register/register.wxml` - 页面结构
- `pages/register/register.js` - 页面逻辑
- `pages/register/register.wxss` - 页面样式
- `pages/register/register.json` - 页面配置

### 2. 云函数
- `cloudfunctions/register/` - 用户注册云函数
- `cloudfunctions/sendSmsCode/` - 发送验证码云函数

### 3. 数据库集合
已创建以下集合:
- `users` - 用户信息
- `sms_codes` - 验证码记录

## 部署步骤

### 方式一: 微信开发者工具部署(推荐)

#### 1. 打开项目
使用微信开发者工具打开 `f:/code/buysomething`

#### 2. 上传云函数

##### 上传注册云函数:
1. 在左侧目录树找到 `cloudfunctions/register`
2. 右键点击 `register` 文件夹
3. 选择"上传并部署:云端安装依赖"
4. 等待上传完成

##### 上传发送验证码云函数:
1. 在左侧目录树找到 `cloudfunctions/sendSmsCode`
2. 右键点击 `sendSmsCode` 文件夹
3. 选择"上传并部署:云端安装依赖"
4. 等待上传完成

#### 3. 配置数据库
在CloudBase控制台创建 `sms_codes` 集合:

```
集合名称: sms_codes
字段说明:
- _id: 自动生成的唯一ID
- phone: 手机号
- code: 验证码
- expiresAt: 过期时间
- createTime: 创建时间
```

#### 4. 测试注册功能

##### 测试流程:
1. 启动小程序
2. 进入登录页面
3. 点击"立即注册"
4. 输入手机号(如: 13800138000)
5. 点击"获取验证码"
6. 输入验证码(当前测试环境固定为: 123456)
7. 设置密码(6-20位)
8. 确认密码
9. 勾选用户协议
10. 点击"立即注册"

##### 预期结果:
- 注册成功后自动返回登录页面
- 可以使用注册的手机号和密码登录
- 数据库 `users` 集合中会新增一条用户记录

### 方式二: 使用CloudBase CLI部署

## 功能说明

### 注册页面特性

#### 1. 表单验证
- **手机号验证**: 必须为11位数字,且符合手机号格式
- **验证码验证**: 必须为6位数字
- **密码验证**: 长度6-20位
- **确认密码验证**: 必须与密码一致

#### 2. 密码强度检测
- **弱**: 密码长度6-7位
- **中**: 包含数字和字母(大小写)
- **强**: 包含数字、大小写字母
- **很强**: 包含数字、大小写字母和特殊字符

#### 3. 验证码发送
- 60秒倒计时,防止重复发送
- 每个手机号5分钟内只能发送一次
- 验证码5分钟有效期

#### 4. 用户协议
- 注册前必须同意用户协议和隐私政策
- 点击链接可查看完整协议内容

### 云函数说明

#### register 云函数

**功能**: 处理用户注册请求

**参数**:
- `phone`: 手机号
- `code`: 验证码
- `password`: 密码

**返回**:
```json
{
  "success": true,
  "message": "注册成功",
  "userInfo": {
    "_id": "用户ID",
    "_openid": "微信OpenID",
    "nickname": "用户昵称",
    "avatar": "头像URL",
    "phone": "手机号",
    "points": 0,
    "level": 1
  }
}
```

**处理逻辑**:
1. 验证手机号格式
2. 验证密码长度
3. 验证验证码(测试环境固定为123456)
4. 检查手机号是否已注册
5. 检查OpenID是否已注册
6. 创建或更新用户记录
7. 返回用户信息

#### sendSmsCode 云函数

**功能**: 发送短信验证码

**参数**:
- `phone`: 手机号

**返回**:
```json
{
  "success": true,
  "message": "验证码已发送",
  "code": "123456" // 测试环境返回,生产环境不返回
}
```

**处理逻辑**:
1. 验证手机号格式
2. 生成6位随机验证码
3. 检查是否有未过期的验证码
4. 存储验证码到数据库(5分钟有效期)
5. 调用腾讯云短信服务发送(需配置)
6. 返回发送结果

## 短信服务配置(可选)

如果需要真实发送短信,需要配置腾讯云短信服务:

### 1. 开通短信服务
访问 [腾讯云短信控制台](https://console.cloud.tencent.com/sms)

### 2. 创建短信应用
- 应用名称: 优选购物
- 应用类型: 小程序

### 3. 创建短信模板
- 模板名称: 注册验证码
- 模板内容: 您的验证码是{1},5分钟内有效。
- 审核通过后获得模板ID

### 4. 修改云函数代码

在 `cloudfunctions/sendSmsCode/index.js` 中:

```javascript
// 取消注释并配置以下内容
const tencentcloud = require("tencentcloud-sdk-nodejs")
const SmsClient = tencentcloud.sms.v20210111.Client

const clientConfig = {
  credential: {
    secretId: "你的腾讯云SecretId",
    secretKey: "你的腾讯云SecretKey",
  },
  region: "ap-guangzhou",
  profile: {
    httpProfile: {
      endpoint: "sms.tencentcloudapi.com",
    },
  },
}

const client = new SmsClient(clientConfig)
const params = {
  PhoneNumberSet: [`+86${phone}`],
  TemplateId: "你的短信模板ID",
  TemplateParamSet: [code],
  SmsSdkAppId: "你的短信应用ID"
}

await client.SendSms(params)
```

### 5. 配置环境变量

在云函数配置中添加环境变量:
- `TENCENT_SECRET_ID`: 腾讯云SecretId
- `TENCENT_SECRET_KEY`: 腾讯云SecretKey

## 测试注意事项

### 测试环境
- 验证码固定为: `123456`
- 可以直接使用此验证码完成注册

### 生产环境
- 需要配置真实的短信服务
- 验证码随机生成

## 常见问题

### 1. 云函数上传失败
**原因**: 网络问题或权限不足

**解决方法**:
- 检查网络连接
- 确认已登录CloudBase
- 尝试重新上传

### 2. 验证码发送失败
**原因**: 未配置短信服务或验证码已过期

**解决方法**:
- 测试环境使用固定验证码 `123456`
- 生产环境需配置腾讯云短信服务
- 检查手机号格式是否正确

### 3. 注册失败
**原因**: 手机号已注册或验证码错误

**解决方法**:
- 检查手机号是否已注册
- 确认验证码是否正确
- 检查密码是否符合要求

### 4. CloudBase初始化失败
**原因**: 环境ID错误或未开通云开发

**解决方法**:
- 确认环境ID正确: `buysomething-6gbmbtpxff05be35`
- 在微信开发者工具中开通云开发
- 检查小程序是否关联到CloudBase环境

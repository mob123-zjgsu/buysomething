# AI 集成心得体会（二）

## 从零到一的突破

当我接到"为小程序添加 AI 智能客服"这个任务时，心中既有期待也有忐忑。作为一名计算机专业的学生，我对 AI 的了解大多停留在理论层面，从未亲手实现过一个真正可用的 AI 应用。这次经历让我实现了从理论到实践的跨越。

## 遇到的挑战与解决

### 挑战一：选择合适的实现方式

最初我在"使用云服务商内置 AI"和"自行调用第三方 API"之间犹豫不决。通过查阅文档、对比成本和稳定性，最终选择了直接调用 DeepSeek API 的方案。这个决策过程让我学会了在众多技术方案中权衡利弊、做出取舍。

### 挑战二：处理网络请求异常

在开发过程中，最常见的问题就是网络请求失败。可能是超时、可能是 API Key 失效、可能是网络不通。为了让用户体验更加友好，我在云函数中添加了完善的错误处理机制，将技术性的错误信息转化为用户友好的提示。这个过程让我深刻体会到"用户体验"不只是一个概念，而是要落实到每一行代码中。

### 挑战三：控制成本与优化性能

AI API 是按调用次数计费的，如何在保证功能的前提下减少不必要的调用，是我需要考虑的问题。通过限制对话历史长度、优化 Prompt 长度等方式，我在功能和成本之间找到了平衡点。这也让我意识到，优秀的工程师不仅要实现功能，更要考虑资源的合理利用。

## 收获与反思

### 技术层面

- 掌握了 HTTPS 请求的发送与接收
- 理解了环境变量在配置管理中的重要作用
- 学会了云函数的开发与部署流程
- 初步了解了 Prompt 工程的基本原则

### 思维层面

- 从"能用就行"到"好用才行"的转变
- 认识到安全意识和工程规范的重要性
- 理解了"需求驱动技术选型"而非"技术驱动需求"

### 未来展望

这次实践让我对 AI 应用开发产生了浓厚的兴趣。我计划在以下几个方面继续深入学习：

1. 探索更多 AI 能力，如内容生成、智能推荐等
2. 学习如何评估和优化 AI 模型的输出质量
3. 关注 AI 伦理和安全问题，成为负责任的 AI 开发者

## 结语

回顾整个 AI 集成过程，从最初的需求分析到最终的部署上线，每一个环节都让我成长不少。感谢这次项目实践，让我有机会将课堂上学到的知识应用到真实场景中，也让我对 AI 技术的未来充满了期待。

我相信，随着技术的不断发展，AI 将会以更多样的形式融入我们的日常生活中。而我，也将带着这份实践经验，继续在 AI 探索的道路上前行。

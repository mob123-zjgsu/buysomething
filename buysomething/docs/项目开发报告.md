## 零、Docker 环境配置（可选）

### 根目录 docker-compose.yml

**路径**: `docker-compose.yml`

**用途**: 用于本地开发环境演示，展示项目架构中包含数据库服务的配置。实际生产环境使用 CloudBase 云端部署，无需本地运行。

**包含服务**:
| 服务 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| mongodb | mongo:6.0 | 27017 | MongoDB 数据库 |
| redis | redis:7-alpine | 6379 | Redis 缓存（可选）|
| api | 自定义 | 3000 | Node.js API 服务 |
| nginx | nginx:alpine | 8080 | 反向代理（可选）|

**使用方式**:
```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

**说明**: 此配置为可选的本地开发环境配置。项目实际使用 CloudBase 云端数据库服务，无需运行此配置即可正常使用小程序。

## 一、组件/模块封装

### 1. product-card（商品卡片组件）

**路径**: `miniprogram/components/product-card/`

**用途**: 展示单个商品的卡片组件，包含图片、价格、销量等信息，支持点击进入详情页、添加购物车、收藏等操作。

**使用方式**:
```wxml
<product-card 
  product="{{productData}}" 
  bind:addcart="onAddCart" 
  bind:favorite="onFavorite"
/>
```

**属性**:
| 属性名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| product | Object | {} | 商品数据对象 |
| showOriginalPrice | Boolean | true | 是否显示原价 |
| showDiscount | Boolean | true | 是否显示折扣标签 |
| showSales | Boolean | true | 是否显示销量 |

**事件**:
| 事件名 | 说明 | 回调参数 |
|--------|------|----------|
| addcart | 点击添加购物车 | 商品数据 |
| favorite | 点击收藏/取消收藏 | 商品数据 |

## 四、遇到的问题和解决

### 前端开发版

#### 问题1：商品图片不显示
**问题描述**: 购物车界面显示的商品图片为空白，商品详情页显示的是外部无效链接。

**原因分析**: 
- 部分商品数据库中存储的是外部图片链接（如 `https://img.example.com/xxx.jpg`）
- 这些外部链接在小程序中无法正常显示

**解决方案**:
1. 在 `product-detail` 云函数中添加图片过滤逻辑，将无效外部链接替换为本地默认图片
2. 在 `loadProductDetail` 函数中单独设置 `product.image` 字段
3. 在添加购物车时优先使用转换后的 `product.image` 而非 `product.images[0]`

#### 问题2：购物车点击商品跳转后商品ID为undefined
**问题描述**: 从购物车点击商品进入详情页时报错 `商品ID: undefined`。

**原因分析**: 
- 从首页添加到购物车的商品没有设置 `selected: true`
- 点击事件未正确传递 `productId`

**解决方案**:
1. 在添加购物车时确保 `selected: true`
2. 添加调试日志定位问题
3. 在点击事件处理函数中添加 ID 检查

#### 问题3：按钮超出屏幕宽度
**问题描述**: 商品详情页的"加入购物车"和"立即购买"按钮超出屏幕。

**解决方案**:
1. 使用 `max-width: 240rpx` 限制按钮最大宽度
2. 调整按钮 `padding` 和 `flex` 比例
3. 使用 `box-sizing: border-box` 避免尺寸溢出

### 后端开发版

#### 问题1：云函数部署后代码未更新
**问题描述**: 修改云函数代码后，小程序调用返回的结果仍然是旧版本。

**原因分析**: 
- 云函数需要手动重新部署才能生效
- 本地代码修改不等于云端已更新

**解决方案**:
1. 修改代码后使用 `tcb fn code update` 或开发者工具重新上传部署
2. 部署后在云函数列表确认更新时间
3. 必要时在控制台重启云函数实例

#### 问题2：数据库查询返回格式不一致
**问题描述**: 不同查询方式返回的数据结构不同（有的直接返回数据，有的返回 `{data: ...}`）。

**解决方案**:
```javascript
// 添加兼容处理
const productData = queryResult.data || queryResult || [];
const product = Array.isArray(productData) ? productData[0] : productData;
```

#### 问题3：NoSQL 数据库删除操作权限问题
**问题描述**: 使用工具删除数据时报错 `update 操作时必须提供 query`。

**解决方案**: 创建专门的 `delete-product` 云函数来处理删除操作，而非直接使用数据库工具的 update 方法。

#### 问题4：云函数超时
**问题描述**: 大数据量查询时云函数响应超时。

**解决方案**:
1. 使用分页查询，限制单次返回数量
2. 添加 `limit` 参数控制
3. 使用云函数并发优化

## 五、心得体会

### 前端开发版

1. **组件化思维**: 将可复用的 UI 部分抽离成独立组件（如 product-card、loading、empty-state），提高代码复用性和可维护性。良好的组件设计应该是高内聚、低耦合的。

2. **数据流管理**: 小程序中 `setData` 是性能关键点，避免频繁调用和深层嵌套的批量更新。同时注意本地存储（Storage）和页面数据状态的同步。

3. **调试技巧**: 添加适当的 `console.log` 日志可以帮助快速定位问题，特别是在云函数调用和数据传递环节。

4. **用户体验**: 加载状态、空状态、错误提示等细节处理直接影响用户体验。好的交互反馈能让用户清楚当前状态。

5. **响应式设计**: 使用 rpx 单位可以较好地适配不同屏幕尺寸，但需要注意边界情况（如按钮溢出、文字截断等）。

### 后端开发版

1. **云函数特性**: 云函数运行在云端，有别于传统服务器，需要注意冷启动、运行超时等特性。合理的超时设置和错误处理至关重要。

2. **NoSQL vs SQL**: 云开发使用的是 NoSQL（MongoDB 风格），与传统 SQL 不同。不需要预先定义表结构，但需要理解文档模型的数据组织方式。

3. **安全规则**: 数据库安全规则控制着数据访问权限，需要合理配置 `read`、`create`、`update`、`delete` 规则，既保证安全性又不影响正常功能。

4. **权限验证**: 小程序使用微信的 openid 机制进行用户识别，比传统 JWT 方案更安全且无需额外开发。理解 openid/unionid 的区别对用户系统设计很重要。

5. **API 设计**: 统一的响应格式（success、message、data）让前端处理更简单。完善的参数验证和错误处理是高质量 API 的基础。

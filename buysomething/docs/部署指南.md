# 🐳 Docker + CloudBase 部署指南

本项目采用混合部署架构：
- **后端**: CloudBase 云函数（无需容器化）
- **前端**: Docker 容器化部署

---

## 📁 项目结构

```
buysomething/
├── cloudfunctions/          # CloudBase 云函数（后端）
│   └── ...                  # 自动部署到 CloudBase
├── frontend/                # 前端演示站（Docker）
│   ├── Dockerfile          # 多阶段构建
│   ├── compose.yaml        # 开发环境
│   ├── compose.prod.yaml   # 生产环境
│   └── .dockerignore       # Docker 忽略文件
├── .github/
│   └── workflows/
│       ├── cloudbase-deploy.yml   # CloudBase 云函数部署
│       └── docker-frontend.yml     # 前端 Docker 镜像构建
└── pages/                   # 微信小程序页面
```

---

## 🔧 必需配置

### 1. CloudBase Secrets（后端部署）

在 GitHub 仓库的 `Settings → Secrets and variables → Actions` 中添加：

| Secret 名称 | 说明 | 获取位置 |
|-------------|------|----------|
| `TCB_SECRET_ID` | 腾讯云 SecretId | [API 密钥管理](https://console.cloud.tencent.com/cam/capi) |
| `TCB_SECRET_KEY` | 腾讯云 SecretKey | [API 密钥管理](https://console.cloud.tencent.com/cam/capi) |
| `TCB_ENV_ID` | CloudBase 环境ID | buysomething-6gbmbtpxff05be35 |

### 2. GitHub Packages（前端 Docker）

自动配置，只需确保 `GITHUB_TOKEN` 有权限（默认已配置）。

## 🚀 部署流程

### 后端：自动部署云函数

1. 推送代码到 `main` 分支的 `cloudfunctions/` 目录
2. GitHub Actions 自动触发 `cloudbase-deploy.yml`
3. 云函数自动部署到 CloudBase

```
推送代码 → GitHub Actions → CloudBase CLI → 云函数部署
```

### 前端：Docker 镜像构建

1. 推送代码到 `main` 分rant 的 `frontend/` 目录
2. GitHub Actions 自动触发 `docker-frontend.yml`
3. Docker 镜像推送到 GHCR

```
推送代码 → GitHub Actions → Docker Build → GHCR 镜像推送
```

## 🐳 本地 Docker 命令

### 前端开发环境

```bash
cd frontend

# 启动开发环境（热重载）
docker compose -f compose.yaml up -d

# 查看日志
docker compose logs -f

# 停止
docker compose down
```

### 前端生产环境

```bash
cd frontend

# 启动生产环境
docker compose -f compose.prod.yaml up -d

# 查看服务状态
docker compose -f compose.prod.yaml ps

# 查看健康检查状态
docker inspect buysomething-frontend-prod --format='{{.State.Health.Status}}'
```

### 手动构建镜像

```bash
cd frontend

# 构建镜像
docker build -t buysomething-frontend:latest .

# 运行容器
docker run -d -p 3000:3000 --name frontend buysomething-frontend:latest

# 测试健康检查
curl http://localhost:3000/health
```

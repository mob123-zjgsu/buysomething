# AI 集成心得体会（一）

## 技术成长

在本次项目中，我有幸将 DeepSeek 大语言模型集成到电商小程序中，实现了智能客服功能。通过这次实践，我对以下方面有了更深入的理解：

### 1. API 调用的理解

从最初对 API 的模糊认知，到能够独立完成从请求构建、参数配置到响应解析的完整流程，我对 RESTful API 的调用方式有了直观的认识。特别是在处理 HTTPS 请求、设置请求头、解析 JSON 响应这些环节，积累了不少实战经验。

### 2. 环境变量安全管理

在集成过程中，我深刻认识到将敏感信息（如 API Key）硬编码在代码中的危害。通过使用环境变量管理 API Key，并配合 `.gitignore` 防止敏感文件泄露，我对信息安全有了更强烈的意识。这不仅适用于 AI 集成，也是所有涉及敏感信息项目的基本原则。

### 3. 云函数开发体验

作为后端服务的新手，我通过云函数开发掌握了事件驱动编程模型，理解了如何从前端传递参数、如何进行错误处理、如何返回统一的响应格式。云函数免去了服务器运维的麻烦，让我能更专注于业务逻辑的实现。

### 4. Prompt 工程初步认识

在与 AI 对话的过程中，我意识到给 AI 清晰的指令（即 Prompt）非常重要。一个好的系统提示词可以让 AI 的回答更加专业、准确。通过不断调整提示词内容，我学会了如何让 AI 更好地理解电商客服这个特定场景。

## 团队协作

虽然本次项目的 AI 功能主要由我负责实现，但在与团队成员沟通需求、讨论实现方案的过程中，我学会了如何将技术方案以非技术人员也能理解的方式表达。同时，我也认识到文档的重要性——一份清晰的技术文档能够大大提高后续维护效率。

## 总结

通过这次 AI 集成项目，我不仅掌握了一项实用的技术能力，更重要的是培养了安全意识和工程思维。AI 技术正在改变软件开发的范式，我很期待在未来的学习和工作中，继续探索 AI 技术的更多应用场景。

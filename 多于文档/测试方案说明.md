# 测试方案说明（教师版）

本文档说明本项目的两种 API 测试方案。

---

## 项目概述

本项目为微信小程序购物应用，使用 CloudBase 云开发，包含以下已部署的云函数：

| 云函数 | 功能 |
|--------|------|
| login | 用户登录 |
| products | 商品列表查询 |
| product-detail | 商品详情查询 |
| cart | 购物车管理 |
| orders | 订单管理 |

- 环境 ID：`buysomething-6gbmbtpxff05be35`
- 测试账号：手机号 `13800138001`，密码 `123456`
- 测试验证码：`0000`

## 方案一：现有页面嵌入测试代码

### 方案描述

在现有小程序页面的 `onLoad` 生命周期中嵌入测试代码，页面加载时自动调用云函数并输出结果到控制台。

### 实现位置

修改文件：`pages/home/home.js`

### 核心代码

```javascript
// ==================== [测试代码开始] ====================
onLoad(options) {
  this.loadMoreProducts()
  
  // 下面的代码用于测试云函数调用
  // 测试完成后可以删除或注释掉
  this.testCloudFunctions()
  // ==================== [测试代码结束] ====================
},

// ==================== [测试代码: 云函数测试] ====================
async testCloudFunctions() {
  console.log('===== 开始云函数测试 =====')
  
  try {
    // 测试1: 用户登录
    const loginRes = await wx.cloud.callFunction({
      name: 'login',
      data: {
        phone: '13800138001',
        password: '123456',
        code: '0000'
      }
    })
    console.log('【测试】登录结果:', loginRes.result)
    
    // 测试2: 获取商品列表
    const productsRes = await wx.cloud.callFunction({
      name: 'products',
      data: { page: 1, pageSize: 10 }
    })
    console.log('【测试】商品列表:', productsRes.result)
    
    // 测试3: 获取商品详情（如果有商品）
    if (productsRes.result?.data?.list?.length > 0) {
      const productId = productsRes.result.data.list[0].productId
      const detailRes = await wx.cloud.callFunction({
        name: 'product-detail',
        data: { productId }
      })
      console.log('【测试】商品详情:', detailRes.result)
      
      // 测试4: 添加购物车
      const cartRes = await wx.cloud.callFunction({
        name: 'cart',
        data: {
          action: 'add',
          userId: 'test-user-001',
          productId,
          quantity: 1
        }
      })
      console.log('【测试】添加购物车:', cartRes.result)
    }
    
    console.log('===== 云函数测试完成 =====')
    
  } catch (err) {
    console.error('【测试】云函数调用失败:', err)
  }
},
// ==================== [测试代码结束] ====================
```

### 方案特点

| 优点 | 缺点 |
|------|------|
| 集成在现有页面中 | 控制台输出，不直观 |
| 无需新建页面 | 可能影响原有页面逻辑 |
| 快速测试 | 需要查看控制台才能看到结果 |

## 方案二：独立测试页面

### 方案描述

创建专门的测试页面，集成所有云函数的测试功能，页面直接显示测试结果和日志。

### 实现位置

新建文件：
- `pages/test/test.js` - 测试逻辑
- `pages/test/test.wxml` - 页面视图
- `pages/test/test.wxss` - 页面样式
- `pages/test/test.json` - 页面配置

并在 `app.json` 中注册页面

### 核心代码

**test.js**（测试逻辑）:
```javascript
Page({
  data: {
    testResults: [],
    logs: []
  },

  onLoad() {
    this.addLog('===== 小程序测试开始 =====')
    this.runAllTests()
  },

  addLog(msg) {
    const timestamp = new Date().toLocaleTimeString()
    this.setData({
      logs: [...this.data.logs, `[${timestamp}] ${msg}`]
    })
    console.log(msg)
  },

  async runAllTests() {
    try {
      this.addLog('开始测试...')
      
      // 测试1: 用户登录
      this.addLog('--- 测试1: 用户登录 ---')
      const loginRes = await wx.cloud.callFunction({
        name: 'login',
        data: {
          phone: '13800138001',
          password: '123456',
          code: '0000'
        }
      })
      this.addLog('登录结果: ' + JSON.stringify(loginRes.result))
      
      // 测试2: 获取商品列表
      this.addLog('--- 测试2: 获取商品列表 ---')
      const productsRes = await wx.cloud.callFunction({
        name: 'products',
        data: { page: 1, pageSize: 10 }
      })
      this.addLog('商品列表结果: ' + JSON.stringify(productsRes.result))
      
      // 测试3: 获取商品详情
      this.addLog('--- 测试3: 获取商品详情 ---')
      const productId = productsRes.result?.data?.list?.[0]?.productId || 'prod-001'
      const detailRes = await wx.cloud.callFunction({
        name: 'product-detail',
        data: { productId }
      })
      this.addLog('商品详情结果: ' + JSON.stringify(detailRes.result))
      
      // 测试4: 添加购物车
      this.addLog('--- 测试4: 添加购物车 ---')
      const cartRes = await wx.cloud.callFunction({
        name: 'cart',
        data: {
          action: 'add',
          userId: 'test-user-001',
          productId,
          quantity: 2
        }
      })
      this.addLog('添加购物车结果: ' + JSON.stringify(cartRes.result))
      
      // 测试5: 获取购物车
      this.addLog('--- 测试5: 获取购物车 ---')
      const cartListRes = await wx.cloud.callFunction({
        name: 'cart',
        data: {
          action: 'list',
          userId: 'test-user-001'
        }
      })
      this.addLog('购物车列表结果: ' + JSON.stringify(cartListRes.result))
      
      // 测试6: 创建订单
      this.addLog('--- 测试6: 创建订单 ---')
      const orderRes = await wx.cloud.callFunction({
        name: 'orders',
        data: {
          action: 'create',
          userId: 'test-user-001',
          items: [{ productId, quantity: 2 }],
          remark: '测试订单'
        }
      })
      this.addLog('创建订单结果: ' + JSON.stringify(orderRes.result))
      
      // 测试7: 获取订单列表
      this.addLog('--- 测试7: 获取订单列表 ---')
      const orderListRes = await wx.cloud.callFunction({
        name: 'orders',
        data: {
          action: 'list',
          userId: 'test-user-001'
        }
      })
      this.addLog('订单列表结果: ' + JSON.stringify(orderListRes.result))
      
      this.addLog('===== 全部测试完成 =====')

    } catch (err) {
      this.addLog('测试失败: ' + JSON.stringify(err))
    }
  },

  refreshTest() {
    this.setData({ testResults: [], logs: [] })
    this.addLog('===== 重新测试开始 =====')
    this.runAllTests()
  }
})
```

**test.wxml**（页面视图）:
```xml
<view class="container">
  <view class="header">
    <text class="title">API 测试页面</text>
    <button class="refresh-btn" bindtap="refreshTest">刷新测试</button>
  </view>

  <scroll-view scroll-y class="log-container">
    <view wx:for="{{logs}}" wx:key="index" class="log-item">
      <text>{{item}}</text>
    </view>
  </scroll-view>

  <view class="result-section">
    <text class="section-title">测试结果 ({{testResults.length}})</text>
    <view wx:for="{{testResults}}" wx:key="name" class="result-item">
      <view class="result-header">
        <text class="result-name">{{item.name}}</text>
        <text class="result-status">{{item.result.code === 0 ? '成功' : '失败'}}</text>
      </view>
      <text class="result-data">{{item.result.message}}</text>
    </view>
  </view>
</view>
```

### 方案特点

| 优点 | 缺点 |
|------|------|
| 页面直接显示测试结果 | 需要新建页面文件 |
| 支持刷新重新测试 | 需在 app.json 注册 |
| 可测试完整的业务流程 | 测试功能独立于业务 |
| 可视化程度高 | |

## 两种方案对比

| 对比项 | 方案一 | 方案二 |
|--------|--------|--------|
| 代码量 | 较少 | 较多 |
| 实现难度 | 简单 | 中等 |
| 结果展示 | 控制台 | 页面显示 |
| 业务流程 | 部分 | 完整 |
| 适用场景 | 快速验证 | 完整测试 |
| 是否需要新页面 | 否 | 是 |

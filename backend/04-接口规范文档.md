## 一、接口设计规范

### 1.1 命名规范

#### 云函数命名

```
{模块名}/{操作名}

示例:
- user/login        // 用户登录
- product/getList   // 获取商品列表
- order/create      // 创建订单
```

#### 数据库集合命名

```
{业务域}_复数形式

示例:
- users      // 用户集合
- products   // 商品集合
- orders     // 订单集合
- cart       // 购物车集合
```

#### 字段命名

- 使用小驼峰命名法(camelCase)
- 时间字段统一后缀 `Time`
- 金额字段后缀 `Amount`
- 数量字段后缀 `Quantity` 或 `Count`

示例:
```javascript
{
  "userId": "user_001",        // 用户ID
  "createTime": "2026-03-24",  // 创建时间
  "totalAmount": 298.00,        // 总金额
  "reviewCount": 200            // 评价数
}
```

---

### 1.2 请求规范

#### 通用请求格式

```javascript
{
  "data": {},        // 业务数据对象
  "timestamp": 1234567890  // 可选,时间戳
}
```

#### 分页参数

```javascript
{
  "page": 1,          // 页码,从1开始
  "pageSize": 20       // 每页数量
}
```

#### 排序参数

```javascript
{
  "sortField": "createTime",  // 排序字段
  "sortOrder": "desc"        // 排序方向: asc升序, desc降序
}
```

#### 筛选参数

```javascript
{
  "filter": {
    "field1": "value1",    // 等于
    "field2": { "$gt": 100 },  // 大于
    "field3": { "$in": [1, 2, 3] }  // 在范围内
  }
}
```

### 1.3 响应规范

#### 成功响应

```json
{
  "code": 0,              // 状态码: 0表示成功
  "message": "success",   // 提示信息
  "data": {              // 业务数据
    // 具体数据内容
  }
}
```

#### 失败响应

```json
{
  "code": 1001,                    // 错误码(非0)
  "message": "参数错误",           // 错误提示
  "data": {}                      // 可选,错误详情
}
```

#### 列表响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],          // 数据列表
    "total": 100,        // 总数
    "page": 1,          // 当前页
    "pageSize": 20       // 每页数量
  }
}
```

### 1.4 状态码规范

#### 系统级状态码 (1xxx)

| 状态码 | message | 说明 |
|-------|---------|------|
| 0 | success | 成功 |
| 1001 | 参数错误 | 请求参数不合法或缺失 |
| 1002 | 未登录 | 用户未登录或登录过期 |
| 1003 | 权限不足 | 无权限访问该资源 |
| 1004 | 资源不存在 | 请求的资源不存在或已删除 |

#### 业务级状态码 (1xxx)

| 状态码 | message | 说明 |
|-------|---------|------|
| 1005 | 商品已下架 | 商品已下架或库存不足 |
| 1006 | 购物车为空 | 购物车中没有商品 |
| 1007 | 订单不存在 | 订单不存在或已删除 |
| 1008 | 余额不足 | 用户余额不足 |
| 1009 | 验证码错误 | 验证码错误或已过期 |
| 1010 | 手机号已注册 | 手机号已被注册 |

#### 系统级错误码 (2xxx)

| 状态码 | message | 说明 |
|-------|---------|------|
| 2001 | 数据库错误 | 数据库操作失败 |
| 2002 | 网络错误 | 网络请求失败 |
| 2003 | 系统繁忙 | 系统繁忙,请稍后重试 |
| 2004 | 云函数错误 | 云函数执行失败 |
| 2005 | 支付失败 | 支付接口调用失败 |

## 二、数据类型规范

### 2.1 基础类型

| 类型 | 说明 | 示例 |
|-----|------|-----|
| String | 字符串 | `"张三"` |
| Number | 数字 | `99.00` |
| Boolean | 布尔值 | `true` / `false` |
| Array | 数组 | `["红色", "蓝色"]` |
| Object | 对象 | `{"color": "白色", "size": "M"}` |
| Date | 日期 | `{"$date": "2026-03-24T10:00:00.000Z"}` |

### 2.2 特殊类型

#### 时间类型

CloudBase数据库使用特殊格式表示日期:

```javascript
{
  "createTime": {
    "$date": "2026-03-24T10:00:00.000Z"
  }
}
```

#### ID类型

主键ID使用字符串类型,建议格式:

```javascript
{
  "_id": "user_001",        // 集合名_序号
  "productId": "product_001",
  "orderId": "order_001"
}
```

## 三、业务字段规范

### 3.1 通用字段

#### 主键字段

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `_id` | String | 是 | 数据库主键,自动生成 |
| `_openid` | String | 是 | 微信OpenID,用户唯一标识 |

#### 时间字段

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `createTime` | Date | 是 | 创建时间 |
| `updateTime` | Date | 是 | 更新时间 |

#### 状态字段

| 字段名 | 类型 | 取值范围 | 说明 |
|-------|------|---------|------|
| `status` | Number | 0-5 | 状态编码,见业务状态表 |

### 3.2 金额字段

所有金额字段使用Number类型,保留2位小数:

```javascript
{
  "price": 99.00,              // 单价
  "totalAmount": 298.00,       // 总金额
  "discountAmount": 29.80,       // 优惠金额
  "payAmount": 268.20          // 实付金额
}
```

### 3.3 数量字段

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `quantity` | Number | 商品数量 |
| `stock` | Number | 库存数量 |
| `sales` | Number | 销量 |
| `totalQuantity` | Number | 总数量 |
| `reviewCount` | Number | 评价数 |

## 六、错误处理规范

### 6.1 云函数错误处理

```javascript
exports.main = async (event, context) => {
  try {
    // 获取用户OpenID
    const wxContext = cloud.getWXContext()
    const openid = wxContext.OPENID
    
    if (!openid) {
      return {
        code: 1002,
        message: '未登录'
      }
    }
    
    // 业务逻辑
    const result = await db.collection('cart').add({
      data: {
        _openid: openid,
        productId: event.productId,
        quantity: event.quantity
      }
    })
    
    return {
      code: 0,
      message: 'success',
      data: {
        cartId: result._id
      }
    }
    
  } catch (err) {
    console.error('云函数执行失败:', err)
    
    return {
      code: 2004,
      message: '云函数执行失败',
      data: {
        error: err.message
      }
    }
  }
}
```

### 6.2 数据库错误处理

```javascript
try {
  const result = await db.collection('products').get()
  return {
    code: 0,
    message: 'success',
    data: result.data
  }
} catch (err) {
  console.error('数据库查询失败:', err)
  return {
    code: 2001,
    message: '数据库查询失败',
    data: {
      error: err.errCode
    }
  }
}
```

## 七、性能优化规范

### 7.1 分页查询

使用 `skip` 和 `limit` 实现分页:

```javascript
const page = event.page || 1
const pageSize = event.pageSize || 20

const result = await db.collection('products')
  .skip((page - 1) * pageSize)
  .limit(pageSize)
  .get()
```

### 7.2 索引使用

确保查询字段有索引:

```javascript
// 创建索引
await db.collection('cart').createIndex({
  keys: {
    _openid: 1,
    productId: 1
  },
  name: 'user_product_index'
})

// 使用索引查询
const result = await db.collection('cart')
  .where({
    _openid: openid,
    productId: productId
  })
  .get()
```

### 7.3 字段投影

只查询需要的字段,减少数据传输:

```javascript
const result = await db.collection('users')
  .field({
    _id: true,
    nickname: true,
    avatar: true
  })
  .get()
```

## 八、安全规范

### 8.1 权限控制

在云函数中验证用户权限:

```javascript
exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext()
  const openid = wxContext.OPENID
  
  // 验证是否为资源所有者
  const resource = await db.collection('orders')
    .doc(event.orderId)
    .get()
  
  if (resource.data._openid !== openid) {
    return {
      code: 1003,
      message: '无权限访问该资源'
    }
  }
  
  // 业务逻辑...
}
```

### 8.2 数据过滤

返回数据前过滤敏感信息:

```javascript
// 不过滤: 返回完整数据
return {
  code: 0,
  message: 'success',
  data: user
}

// 过滤: 隐藏敏感字段
const { phone, ...safeUser } = user
return {
  code: 0,
  message: 'success',
  data: safeUser
}
```

### 8.3 SQL注入防护

使用CloudBase提供的查询方法,避免字符串拼接:

```javascript
// ❌ 危险: 字符串拼接
const result = await db.collection('products')
  .where(`name = '${event.keyword}'`)
  .get()

// ✅ 安全: 参数化查询
const result = await db.collection('products')
  .where({
    name: event.keyword
  })
  .get()
```

## 九、日志规范

### 9.1 日志级别

| 级别 | 用途 | 示例 |
|------|------|-----|
| `console.log` | 普通日志 | `console.log('用户登录成功', openid)` |
| `console.warn` | 警告日志 | `console.warn('库存不足', productId)` |
| `console.error` | 错误日志 | `console.error('数据库错误', err)` |

### 9.2 日志内容

日志应包含:时间、操作、参数、结果:

```javascript
console.log({
  time: new Date().toISOString(),
  action: 'cart/add',
  openid: openid,
  params: event,
  result: 'success'
})
```

### 9.3 错误日志

记录完整的错误信息:

```javascript
try {
  // 业务逻辑
} catch (err) {
  console.error({
    time: new Date().toISOString(),
    action: 'order/create',
    openid: openid,
    error: {
      message: err.message,
      stack: err.stack,
      errCode: err.errCode
    }
  })
}
```

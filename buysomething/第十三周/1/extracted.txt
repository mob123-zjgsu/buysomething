作业：云服务部署
作业目标
将项目部署到云服务平台，实现从本地开发到线上访问的完整流程。
具体要求
1. 平台选择
选择以下任一平台进行部署：
平台 特点 适合场景
Vercel 免费、自动部署 前端/全栈
Railway 简单、支持 Docker 后端服务
Render 免费层、易用 全栈应用
阿里云/腾讯云 国内访问快 生产环境
2. 部署配置
根据平台要求配置部署：
# Vercel (vercel.json)
{
"builds": [
{ "src": "frontend/**", "use": "@vercel/static" }
]
}
# Railway / Render
# 使用 Dockerfile 或 nixpacks 自动检测
3. 环境变量配置
在云平台配置环境变量：
• DATABASE_URL - 数据库连接
• API_KEY - 第三方服务密钥
• NODE_ENV=production

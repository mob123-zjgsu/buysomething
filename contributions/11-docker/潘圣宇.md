## 遇到的问题和解决

1. 问题：CloudBase CLI 在 GitHub Actions 中认证失败
   解决：在 GitHub Secrets 中配置 `TCB_SECRET_ID`、`TCB_SECRET_KEY`、`TCB_ENV_ID`，使用 `cloudbase/tencentcloudbase-github-action` 进行认证

2. 问题：每次推送都部署所有云函数，效率低下
   解决：使用 `git diff` 检测变更的云函数，实现增量部署，大幅减少部署时间

## 心得体会


虽然本项目后端没有直接使用 Docker，但通过 CloudBase 云函数的学习，我理解了 Serverless 架构的优势。云函数作为后端服务，无需关心服务器运维、扩容、负载均衡等基础设施问题，只需专注于业务逻辑开发。GitHub Actions 与 CloudBase 的结合实现了完整的自动化部署流程，代码推送后自动部署到云端，大幅提高了开发效率。同时，我也了解到 Docker 在传统后端服务中的重要性，以及 Serverless 和容器化两种架构的适用场景差异。

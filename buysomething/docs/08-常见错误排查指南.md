## 1. 云函数相关错误

### 错误1: FunctionName parameter could not be found

**错误信息:**
```
Error: cloud.callFunction:fail Error: errCode: -501000
| errMsg: FunctionName parameter could not be found
```

**原因:**
云函数未部署或函数名称错误。

**解决方案:**

#### 方案1: 检查云函数是否已部署

```bash
# 使用CloudBase CLI查看云函数列表
tcb functions:list
```

或在微信开发者工具中：
- 右键 `cloudfunctions/register` 文件夹
- 选择"上传并部署:云端安装依赖"

#### 方案2: 使用命令行创建云函数

```bash
# 进入项目目录
cd f:/code/buysomething

# 使用CloudBase工具创建云函数
tcb functions:deploy register
```

#### 方案3: 在CloudBase控制台创建

1. 访问 [CloudBase云函数控制台](https://tcb.cloud.tencent.com/dev?envId=buysomething-6gbmbtpxff05be35#/scf)
2. 点击"新建函数"
3. 选择"自定义函数"
4. 填写函数信息：
   - 函数名称：register
   - 运行环境：Nodejs16.13
   - 提交方式：在线编辑
5. 复制代码到编辑器

**当前状态:**
✅ `register` 云函数已成功部署（2026-03-24 23:03:49）

### 错误2: 云函数调用超时

**错误信息:**
```
Error: cloud.callFunction:fail Error: errCode: -504000
| errMsg: The function execution time exceeded the limit
```

**原因:**
云函数执行时间超过了设置的超时时间（默认60秒）。

**解决方案:**

1. 优化云函数代码，减少执行时间
2. 增加云函数超时时间（最多900秒）

```javascript
// 在创建云函数时设置超时时间
{
  "name": "register",
  "timeout": 120  // 设置为120秒
}
```

3. 检查是否有死循环或阻塞操作

---

### 错误3: 云函数内存不足

**错误信息:**
```
Error: cloud.callFunction:fail Error: errCode: -502003
| errMsg: The function process ran out of memory
```

**原因:**
云函数使用的内存超过了分配的内存限制。

**解决方案:**

1. 增加云函数内存配置（最大3GB）

```javascript
{
  "name": "register",
  "memorySize": 512  // 设置为512MB
}
```

2. 优化代码，减少内存使用

## 2. 数据库相关错误

### 错误1: 集合不存在

**错误信息:**
```
Error: cloud.database.command failed: Error: Collection not found
```

**原因:**
数据库集合未创建。

**解决方案:**

#### 方案1: 在控制台创建集合

1. 访问 [CloudBase数据库控制台](https://tcb.cloud.tencent.com/dev?envId=buysomething-6gbmbtpxff05be35#/db/doc)
2. 点击"添加集合"
3. 输入集合名称（如：users）
4. 点击"确定"

#### 方案2: 使用云函数创建集合

```javascript
const cloud = require('wx-server-sdk')
const db = cloud.database()

exports.main = async (event, context) => {
  try {
    // 创建多个集合
    await db.createCollection('users')
    await db.createCollection('products')
    await db.createCollection('categories')
    await db.createCollection('cart')
    await db.createCollection('orders')
    await db.createCollection('order_items')
    await db.createCollection('addresses')
    await db.createCollection('coupons')
    await db.createCollection('favorites')
    await db.createCollection('banners')

    return { success: true, message: '集合创建成功' }
  } catch (err) {
    return { success: false, message: err.message }
  }
}
```

**当前状态:**
✅ 已创建10个数据库集合

### 错误2: 权限不足

**错误信息:**
```
Error: cloud.database.command failed: Error: Permission denied
```

**原因:**
数据库安全规则配置不正确，用户没有访问权限。

**解决方案:**

1. 检查数据库安全规则
2. 修改安全规则，允许用户访问

```json
// users 集合安全规则
{
  "read": "auth.openid == doc.openid",
  "write": "auth.openid == doc.openid"
}
```

3. 确保用户已登录（需要OpenID）

---

### 错误3: 权限检查失败

**错误信息:**
```
Error: cloud.database.command failed: Error: auth.uid is missing
```

**原因:**
用户未登录，无法获取OpenID。

**解决方案:**

1. 确保已初始化CloudBase

```javascript
// app.js
wx.cloud.init({
  env: 'buysomething-6gbmbtpxff05be35',
  traceUser: true
})
```

2. 调用云函数获取OpenID

```javascript
wx.cloud.callFunction({
  name: 'getOpenId',
  success: (res) => {
    console.log('OpenID:', res.result.openid)
    wx.setStorageSync('openid', res.result.openid)
  }
})
```

## 3. 前端相关错误

### 错误1: TypeError: Cannot read property '__subPageFrameEndTime__' of null

**错误信息:**
```
TypeError: Cannot read property '__subPageFrameEndTime__' of null
```

**原因:**
定时器在页面卸载后没有正确清除。

**解决方案:**

1. 在页面中添加定时器清理逻辑

```javascript
Page({
  data: {
    countdown: 0
  },

  // 保存定时器引用
  countdownTimer: null,

  startCountdown() {
    let countdown = 60
    this.setData({ countdown })

    // 清除之前的定时器
    if (this.countdownTimer) {
      clearInterval(this.countdownTimer)
    }

    this.countdownTimer = setInterval(() => {
      countdown--
      this.setData({ countdown })

      if (countdown <= 0) {
        clearInterval(this.countdownTimer)
        this.countdownTimer = null
      }
    }, 1000)
  },

  // 页面卸载时清除定时器
  onUnload() {
    if (this.countdownTimer) {
      clearInterval(this.countdownTimer)
      this.countdownTimer = null
    }
  }
})
```

2. 重启微信开发者工具
3. 清除缓存后重新编译

**当前状态:**
✅ 已修复 `login.js` 和 `register.js` 中的定时器问题

### 错误2: 页面跳转失败

**错误信息:**
```
Error: navigateTo:fail page "pages/xxx/xxx" is not found
```

**原因:**
页面未在 app.json 中注册。

**解决方案:**

1. 在 app.json 的 pages 数组中注册页面

```json
{
  "pages": [
    "pages/splash/splash",
    "pages/home/home",
    "pages/register/register",  // 确保页面路径正确
    "pages/login/login"
  ]
}
```

2. 检查页面路径是否正确
3. 检查跳转方法是否正确（switchTab/navigateTo）

```javascript
// 正确的跳转方法
wx.navigateTo({
  url: '/pages/register/register'
})

// 如果是TabBar页面，使用switchTab
wx.switchTab({
  url: '/pages/home/home'
})
```

### 错误3: wx is not defined

**错误信息:**
```
ReferenceError: wx is not defined
```

**原因:**
在非小程序环境中使用了微信API。

**解决方案:**

1. 确保在微信开发者工具中运行
2. 检查是否在云函数中使用了前端API（云函数中使用 `cloud` 而不是 `wx`）

```javascript
// 云函数中
const cloud = require('wx-server-sdk')
const db = cloud.database()  // 正确

// 云函数中不要使用
wx.cloud.database()  // 错误
```

## 4. 环境配置错误

### 错误1: 请使用 2.2.3 或以上的基础库以使用云能力

**错误信息:**
```
console.error('请使用 2.2.3 或以上的基础库以使用云能力')
```

**原因:**
基础库版本过低。

**解决方案:**

1. 更新微信开发者工具到最新版本
2. 更新小程序基础库版本

```
微信开发者工具 → 详情 → 本地设置 → 调试基础库
选择 2.24.0 或更高版本
```

3. 检查 app.json 中的配置

```json
{
  "libVersion": "2.19.4"  // 确保版本 >= 2.2.3
}
```

---

### 错误2: 环境ID错误

**错误信息:**
```
Error: cloud init failed: Invalid env id
```

**原因:**
CloudBase环境ID配置错误。

**解决方案:**

1. 检查环境ID是否正确
2. 获取正确的环境ID

```
云开发控制台 → 设置 → 环境设置 → 环境ID
```

3. 更新 app.js 中的环境ID

```javascript
wx.cloud.init({
  env: 'buysomething-6gbmbtpxff05be35',  // 确保环境ID正确
  traceUser: true
})
```

**当前环境ID:** `buysomething-6gbmbtpxff05be35`

### 错误3: URL校验失败

**错误信息:**
```
request:fail url not in domain list
```

**原因:**
域名未在小程序后台配置。

**解决方案:**

1. 在开发环境中关闭域名检查

```
微信开发者工具 → 详情 → 本地设置 → 不校验合法域名
```

2. 在生产环境中配置服务器域名

```
小程序后台 → 开发 → 开发管理 → 开发设置 → 服务器域名
```

添加以下域名：
- request合法域名：your-api-domain.com
- uploadFile合法域名：your-upload-domain.com
- downloadFile合法域名：your-download-domain.com

## 快速检查清单

遇到错误时，按以下顺序检查：

1. **环境配置**
   - [ ] 环境ID是否正确
   - [ ] 基础库版本是否 >= 2.2.3
   - [ ] CloudBase是否已初始化

2. **云函数**
   - [ ] 云函数是否已部署
   - [ ] 函数名称是否正确
   - [ ] 云函数是否处于活跃状态

3. **数据库**
   - [ ] 集合是否已创建
   - [ ] 安全规则是否配置正确
   - [ ] 用户是否已登录

4. **前端代码**
   - [ ] API调用是否正确
   - [ ] 页面路径是否在app.json中注册
   - [ ] 定时器是否正确清理

5. **网络**
   - [ ] 是否关闭了域名检查（开发环境）
   - [ ] 服务器域名是否配置（生产环境）
